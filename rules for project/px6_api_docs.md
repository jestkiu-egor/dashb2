# Документация px6.link API

Обращение к API производится по адресу: `https://px6.link/api/{api_key}/{method}/?{params}`

## Основные параметры
- **api_key**: Секретный ключ из личного кабинета.
- **method**: Название метода.
- **params**: Дополнительные параметры.

## Лимиты
Не более 3-х запросов в 1 секунду. При превышении — ошибка 429.

## Базовый формат ответа
Успех: `{"status": "yes", "user_id": "...", "balance": "...", "currency": "..."}`
Ошибка: `{"status": "no", "error_id": 100, "error": "Описание"}`

## Методы
### getproxy
Получение списка ваших прокси.
Параметры: `state` (active, expired, expiring, all), `descr`, `nokey`, `page`, `limit`.

**Формат списка:**
Объект `list`, где ключи — внутренние ID. Поля каждого элемента:
- `id`: Внутренний номер.
- `ip`: IPv4 или IPv6.
- `host`: IPv4 (если есть).
- `port`: Порт.
- `user`: Логин.
- `pass`: Пароль.
- `unixtime_end`: Дата окончания (seconds).
- `active`: Статус (1/0).

### buy
Покупка прокси.
Параметры: `count`, `period`, `country`, `version` (4, 3, 6).

### prolong
Продление прокси.
Параметры: `period`, `ids` (через запятую).

### check
Проверка валидности.
Параметры: `ids` или `proxy` (ip:port:user:pass).
