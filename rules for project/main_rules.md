# Правила проекта

1. **Мониторинг правил**: Всегда следить за этой папкой (`rules for project`) и её файлами. Обновлять их по мере работы над проектом.
2. **Перезапуск деплоя**: Если деплой на Vercel не запустился автоматически или возникла ошибка проверки, использовать пустой коммит (`git commit --allow-empty`) для повторного триггера.
3. **Основной репозиторий**: `https://github.com/jestkiu-egor/dashb2.git`. Основная рабочая ветка: `kortez_vetka`.
4. **Оформление коммитов**: ВАЖНО ПИСАТЬ КОМИТЫ НА РУССКОМ ЯЗЫКЕ. Каждый коммит должен содержать заголовок и ПОДРОБНОЕ ОПИСАНИЕ по схеме: "Проблема: ... Как решили: ...".
5. **Интеграция px6.link API**:
   - Base URL: `https://px6.link/api/{api_key}/{method}/`
   - Успешный статус: `"status": "yes"`
   - Метод получения прокси: `getproxy`
   - Формат списка: Объект `list`, где ключи - ID прокси.
   - Поля: `host` (IPv4), `port`, `user` (login), `pass` (password), `unixtime_end` (дата окончания).
