# Правила проекта

1. **Мониторинг правил**: Всегда следить за этой папкой (`rules for project`) и её файлами. Обновлять их по мере работы над проектом.
2. **Перезапуск деплоя**: Если деплой на Vercel не запустился автоматически или возникла ошибка проверки, использовать пустой коммит (`git commit --allow-empty`) для повторного триггера.
3. **Основной репозиторий**: `https://github.com/jestkiu-egor/dashb2.git`. Основная рабочая ветка: `jestkiu_vetka`.
4. **Оформление коммитов**: ВАЖНО ПИСАТЬ КОМИТЫ НА РУССКОМ ЯЗЫКЕ. Каждый коммит должен содержать заголовок и ПОДРОБНОЕ ОПИСАНИЕ по схеме: "Проблема: ... Как решили: ...".
5. **Merge в main**: При слиянии веток в main обязательно писать на русском языке с детальным описанием изменений. Указывать какие функции/файлы добавлены, изменены или удалены. Сравнить основные отличия ветки main и той ветки, с которой идёт merge.
6. **Запрет на push в main**: Напрямую пушить в ветку main ЗАПРЕЩЕНО. Пушить в main можно ТОЛЬКО через merge из другой ветки с разрешения.
7. **Запрет на push без разрешения**: ПРИНУДИТЕЛЬНО не запускать пуш ни в какую ветку без явного сообщения от пользователя. Пушить только после получения явного сигнала от пользователя.
8. **Коммит и пуш только по команде**: Делать коммит и пуш только после получения команды от пользователя. Самостоятельно не инициировать коммит или пуш.
9. **Работа только с jestkiu_vetka**: Работать только с веткой `jestkiu_vetka`. Не создавать новых веток и тегов.
10. **Интеграция px6.link API**:
   - Base URL: `https://px6.link/api/{api_key}/{method}/`
   - Успешный статус: `"status": "yes"`
   - Метод получения прокси: `getproxy`
   - Формат списка: Объект `list`, где ключи - ID прокси.
   - Поля: `host` (IPv4), `port`, `user` (login), `pass` (password), `unixtime_end` (дата окончания).
